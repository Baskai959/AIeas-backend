## 4. 未完成任务清单

按当前代码与文件结构判断，以下事项仍未完成或至少未形成可验收闭环：

1. **敏感配置治理**
   - 移除仓库配置文件中的真实 DSN、Redis 密码等敏感信息，改为 `.env`、环境变量、密钥管理或本地模板。
   - 禁止生产默认 JWT secret 使用 `change-me-in-prod`。

2. **Dockerfile / docker-compose / deployments**
   - 当前未发现 `Dockerfile`、`deployments/`、`docker-compose.yaml/yml`。
   - 需要补齐 MySQL、Redis、服务端、迁移执行与本地演示启动链路。

3. **`/healthz`、`/readyz` 与真正 Prometheus metrics**
   - 当前只有 `/ping` 和占位 `/metrics`。
   - 需要区分存活检查、依赖就绪检查、Lua SHA 加载状态、worker 状态，并接入 Prometheus 指标。

4. **Makefile / CI**
   - 当前未发现 `Makefile` 或 `.github/workflows/*`。
   - 需要标准化 `fmt`、`test`、`lint`、`build`、`migrate`、`openapi` 校验流程。

5. **Redis Stream / 跨实例事件补偿**
   - 当前补偿依赖单进程 Hub 历史窗口。
   - 多实例部署下需要 Redis Stream 或等价事件日志支撑 WS 重连补偿和跨实例广播。

6. **多实例在线人数 Redis 化**
   - 当前在线人数来自 Hub 内存房间。
   - 多实例需要 Redis Set/Hash + TTL 心跳聚合，并处理实例异常退出。

7. **异步 `bid_record` worker 与补偿**
   - 当前 `persistBid` 同步写入，失败只吞掉非冲突错误。
   - 需要有界队列、批量写库、失败持久化、指标告警与补偿重放。

8. **应用重启恢复运行中拍卖**
   - 当前启动时没有扫描 MySQL `RUNNING/EXTENDED` 拍卖并恢复 Redis 热状态和 timer 的代码。

9. **cron 兜底落锤**
   - 当前是拍卖启动时的单实例 timer。
   - 需要全局扫描已到期但未关闭拍卖，避免 timer 丢失或应用重启导致无法自动落锤。

10. **后台配置 / 看板 / 风控规则**
    - 数据库有 `config_item`，文档有看板和风控规划；当前代码只覆盖黑名单和风险事件处理。
    - 需要配置项 CRUD、看板指标查询、风控规则管理与生效机制。

11. **支付超时 / 退款 / 取消**
    - 当前订单状态枚举已有 `TIMEOUT/CANCELLED`、支付状态已有 `REFUNDED`，但服务逻辑未实现对应流程。

12. **真实 MySQL / Redis 完整链路联调**
    - 已有 `AIEAS_INTEGRATION=1` smoke 测试入口，但默认跳过。
    - 需要在真实环境跑通「迁移 → seed → 登录 → 商品/拍品 → 报名 → WS 出价 → 落锤 → 订单支付 → 后台查询」。

13. **压测脚本与验收报告**
    - 当前未发现 k6、vegeta 或自研压测脚本。
    - 需要覆盖单房间高并发出价、同价排序、延时、落锤一致性和 Redis/MySQL 对账。

14. **AI 加分项**
    - 文档/OpenAPI 包含 AI 起拍价建议、公告等接口规划，但当前服务路由未注册 `/api/v1/ai/*`。

## 5. 主要风险与技术债

1. **敏感信息泄露风险最高**：`configs/config.yaml` 中含真实连接信息和口令字段，需优先处理；本文不复述具体值。
2. **单实例假设较强**：WebSocket Hub、在线人数、lastSeq 补偿、timer 都是进程内状态；横向扩容后会出现事件不可见、在线人数不准、自动落锤遗漏等问题。
3. **出价持久化可靠性不足**：当前 `persistBid` 对非冲突写入错误直接返回且不暴露补偿路径，见 `internal/service/bid.go:145-166`。
4. **监控不可用于生产**：`/metrics` 当前只是 JSON 占位，不是 Prometheus 指标；也缺少业务链路指标、Lua 延迟、出价拒绝原因、worker 积压等观测项。
5. **支付状态机未闭环**：支付成功已完成，但超时、取消、退款、回调幂等和保证金/订单联动仍未实现。
6. **文档接口多于实际代码**：OpenAPI/API 文档中包含 AI、看板、健康监控等规划项，当前代码尚未全部注册对应路由，后续需要对文档标注「已实现/规划中」或补齐实现。
7. **真实环境验证不足**：真实 MySQL/Redis 测试入口存在但默认跳过，缺少可重复的一键联调环境和验收报告。
