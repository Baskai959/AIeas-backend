## 4. Logs（slog）

- **JSON**（`OBSERVABILITY_FORMAT=json`）：`time / level / msg` + 自动 `trace_id` / `span_id`（仅当 ctx 中存在 active span，`tracelogger.go:33-39`） + 业务 attrs
- **Text**（默认）：彩色对齐，TTY 启用 ANSI256，**不注入 trace_id**（避免行宽爆炸）
- **GORM 慢 SQL**：阈值 `OBSERVABILITY_SLOW_SQL_THRESHOLD_MS`（默认 200ms，<=0 关闭）；字段 `sql / rows / elapsed_ms / source / error / slow=true`、`component=gorm`

## 7. 我如何查看？

### 7.1 直接 curl

```bash
curl -s http://localhost:8888/metrics | head -50
# 有 token 时：
curl -s -H "Authorization: Bearer $METRICS_TOKEN" http://localhost:8888/metrics | head -50
curl -s http://localhost:8888/readyz | jq .
curl -s http://localhost:8888/healthz
```

### 7.2 本地 Prometheus 抓取

最小 `prometheus.yml`：

```yaml
global:
  scrape_interval: 5s
scrape_configs:
  - job_name: aieas_backend
    metrics_path: /metrics
    static_configs:
      - targets: ["127.0.0.1:8888"]
    # 仅当配了 token 时：
    authorization:
      type: Bearer
      credentials: "${METRICS_TOKEN}"
```

然后 `prometheus --config.file=prometheus.yml`，浏览器开 `http://localhost:9090`。

### 7.3 本地 OTLP + Jaeger 看 traces

`docker-compose.yml`：

```yaml
services:
  jaeger:
    image: jaegertracing/all-in-one:1.56
    ports: ["16686:16686","4318:4318","4317:4317"]
    environment: [COLLECTOR_OTLP_ENABLED=true]
```

启动后服务侧：

```bash
export OBSERVABILITY_TRACING_ENABLED=true
export OBSERVABILITY_TRACING_EXPORTER=otlphttp
export OBSERVABILITY_TRACING_ENDPOINT=localhost:4318   # 注意不带 http://
export OBSERVABILITY_TRACING_INSECURE=true
export OBSERVABILITY_TRACING_SAMPLER=always_on         # 调试期建议全采
go run .
```

打开 `http://localhost:16686`，service 选 `aieas-backend`，可看到 `bid.place` / `hammer.close` / `bid_record.consume` + HTTP / Redis / GORM span 全链路。

### 7.4 看日志

```bash
# JSON：可直接接 jq / Loki / ELK
OBSERVABILITY_FORMAT=json go run . 2>&1 | jq .
# 按 trace 串联（trace_id 来自 X-Trace-Id 响应头或 Jaeger）
OBSERVABILITY_FORMAT=json go run . 2>&1 | jq 'select(.trace_id=="<id>")'
# 本地开发 text 模式（彩色，不注 trace_id）
OBSERVABILITY_FORMAT=text go run .
```

慢 SQL 出现为 `WARN msg="gorm slow sql"`，含 `sql / elapsed_ms / rows / source`。

## 8. 想加新指标 / span？

- **新指标**：在 `metrics/registry.go` `register()` 追加 NewCounterVec / NewHistogramVec → 写入 `collectors` 切片 → 在 `observers.go` 暴露 `Observe*` / `Inc*` → 调用方注入 `*metrics.Registry` 调用即可（nil-safe）。
- **新 span**：`tracing.StartSpan(ctx, "domain.op", attrs...)` + `defer span.End()`；跨进程载体用 `tracing.InjectMap` / `ExtractMap`（参考 bid 链路）。
- 部分 observer（precheck reject / stream xadd / mysql query / worker DLQ / ws drop / agent task 等）已注册指标但调用方较少，是后续业务侧补埋的预留入口。

## 9. 注意事项

- `configs/config.yaml:69` 当前 `logLevel: "dev"` 不在 `debug / info / warn / error` 集合内，会回退为 `info`，建议改为 `info` 或 `debug`。
- yaml 与 `Default()` 在 `sampleRatio` 上不同（1.0 vs 0.1）：实际取值按 yaml < `.env` < env var 顺序覆盖。
- `tracing.buildExporter` 当前**只实现 otlphttp**，Validate 虽接受 otlpgrpc / stdout / noop 字面量，启用前请确保 exporter 选 `otlphttp`，否则会回退 noop。
