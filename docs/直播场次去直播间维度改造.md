# 直播场次去直播间维度改造说明

## 背景

本次改造后，直播拍卖主链路只保留 `live_session` 维度。商家与直播场次是一对多关系，开播、下播、挂载拍品、取消讲解、重新开拍、统计、封面、AI 直播助手 Hook 与 WebSocket 都以直播场次 ID 作为唯一业务上下文。

旧的 `live_room` 维度不再对前后端公开，也不再保留数据库物理结构。

## 已删除接口

- `GET /api/v1/live-rooms`
- `POST /api/v1/live-rooms`
- `GET /api/v1/live-rooms/{id}`
- `PATCH /api/v1/live-rooms/{id}`
- `DELETE /api/v1/live-rooms/{id}`
- `POST /api/v1/live-rooms/{id}/activate`
- `POST /api/v1/live-rooms/{id}/deactivate`
- `GET /api/v1/live-rooms/{id}/lots`
- `POST /api/v1/live-rooms/{id}/lots`
- `DELETE /api/v1/live-rooms/{id}/lots/{auctionId}`
- `GET /api/v1/live-rooms/{id}/stats`
- `GET /ws/live-rooms/{roomId}`

## 替代接口

- `GET /api/v1/live-sessions`：查询商家直播场次列表。
- `POST /api/v1/live-sessions`：创建直播场次。
- `GET /api/v1/live-sessions/{id}`：查询直播场次详情。
- `PATCH /api/v1/live-sessions/{id}`：编辑直播场次基础信息。
- `POST /api/v1/live-sessions/{id}/start`：开播。
- `POST /api/v1/live-sessions/{id}/end`：下播。
- `POST /api/v1/live-sessions/{id}/cover`：上传并写回直播场次封面。
- `GET /api/v1/live-sessions/{id}/lots`：查询场次拍品。
- `POST /api/v1/live-sessions/{id}/lots`：将拍品挂载到场次。
- `DELETE /api/v1/live-sessions/{id}/lots/{auctionId}`：将未成交拍品从场次下架。
- `POST /api/v1/live-sessions/{id}/activate`：开始讲解并开拍指定拍品。
- `POST /api/v1/live-sessions/{id}/deactivate`：取消讲解，拍品可重新开拍或下架。
- `GET /api/v1/live-sessions/{id}/stats`：查询场次统计。
- `GET /api/v1/live-sessions/{id}/agent-hook`：查询 AI 直播助手开关。
- `PATCH /api/v1/live-sessions/{id}/agent-hook`：开启或关闭 AI 直播助手。
- `GET /ws/live-sessions/{sessionId}`：连接直播场次实时消息。

完整契约见 `docs/API/直播场次去直播间维度.openapi.json`。

## 字段变更

- 请求和响应不再出现 `liveRoomId`。
- 拍品挂载关系统一使用 `liveSessionId`。
- `AuctionLot` 不再携带 `liveRoomId`。
- `LiveSession` 不再携带 `liveRoomId`。
- 未开播场次的 `openedAt` 返回 `null` 或不返回，不再返回零值时间。
- Redis 活拍锁语义改为直播场次：`live_session:{id}:active`。

## Agent Hook

所有 Hook 事件都以直播场次 ID 触发：

- 开播：`直播场次{liveSessionId}开播了`
- 拍品上架：`直播场次{liveSessionId}的{auctionId}上架了`
- 拍品下架：`直播场次{liveSessionId}的{auctionId}下架了`
- 最高价出价：`直播场次{liveSessionId}用户{bidderId}目前最高价格{price}`
- 落锤成交：`直播场次{liveSessionId}的{auctionId}落锤成交，成交价{price}`
- 开关开启：`直播场次{liveSessionId}AI直播助手已开启`

关闭 AI 直播助手不发送 Hook。

## 数据库迁移

新增迁移 `migrations/00015_drop_live_room_dimension.sql`。

Up 迁移流程：

- 先基于历史 `live_session.live_room_id` 回填 `auction_lot.live_session_id`。
- 再按拍品归属回填 `bid_record.live_session_id` 和 `order_deal.live_session_id`。
- 删除 `auction_lot.live_room_id`、`live_session.live_room_id` 及对应索引。
- 删除 `live_room` 表。

Down 迁移只恢复结构：重建空 `live_room` 表，并恢复 `auction_lot.live_room_id`、`live_session.live_room_id` 字段及索引。历史直播间业务数据不可恢复。

## 前端迁移点

- 所有页面跳转切到 `/live-sessions`。
- WebSocket 只连接 `/ws/live-sessions/{sessionId}`。
- 服务、类型、store、页面目录切到 `liveSession` 命名。
- `/live-sessions` 展示商家的直播场次列表，不再自动取第一条场次跳转，也不自动创建场次。
- 待上架拍品按 `liveSessionId` 过滤：已挂载到任意场次的拍品不再出现在可挂载列表。
