# 拍品去商品维度变更说明

## 删除/废弃旧商品接口

- 公开路由不再注册 `/api/v1/items`、`/api/v1/items/:id`、`/api/v1/items/audit/callback`。
- 商品描述优化入口迁移为 `/api/v1/auctions/description/optimize`。
- `item` 表通过 migration `00017_collapse_item_into_auction_lot.sql` 重命名为 `legacy_item` 保留回滚数据，主链路不再读取。

## 新拍品创建/编辑/列表字段

拍品创建/编辑直接传展示字段与拍卖规则字段：`title`、`description`、`category`、`brand`、`condition`、`imageUrls`、`coverUrl`、`startPrice`、`reservePrice`、`capPrice`、`incrementRule`、`depositAmount`、`durationSec`、`startTime`、`endTime`。商家只允许写入 `DRAFT` 或 `PENDING_AUDIT` 状态；发布拍品后进入 `PENDING_AUDIT`，后台异步调用 AI 审核，AI 通过后由回调推进为 `READY`，AI 驳回后推进为 `CLOSED_FAILED`。

拍品列表/详情返回上述展示字段，不再返回 `itemId`。直播场次挂载仍只传 `auctionId`。

## 前端迁移点

- 菜单和路由移除商品管理入口。
- 创建/编辑拍品页新增拍品展示信息填写区，AI 描述优化改为拍品语义。
- 拍品列表、详情、直播工作台/控场、直播记录直接读取拍品字段。
- 订单列表/详情读取 `lotSnapshot` 展示成交时拍品快照，避免后续编辑影响历史订单。

## 数据库字段变更

- `auction_lot` 新增 `title`、`description`、`category`、`brand`、`condition_grade`、`image_urls`、`cover_url`。
- 按 `auction_lot.item_id = item.id` 回填展示字段后删除 `auction_lot.item_id` 与 `idx_item_id`。
- `item` 重命名为 `legacy_item`。
- `order_deal` 新增 `lot_snapshot JSON`，历史订单按当前拍品数据回填。
